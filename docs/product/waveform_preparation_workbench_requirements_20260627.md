# 数据准备交互式波形工作台需求文档

版本：2026-06-27
范围：QLanalyser 主工作台的数据准备页；不包含癫痫源码工作台复刻。

## 1. 背景

当前数据准备页已有波形预览、滤波、参考、坏道和片段处理入口，但用户看到的体验仍容易被理解为“静态预览”。数据准备阶段的真实任务不是看一张图，而是在波形上下文中完成预处理决策，并将这些决策保存为可追溯的数据准备记录。

## 2. 用户目标

科研人员在分析前需要：

1. 单击数据后立即看到 EEG 波形。
2. 在同一页面、同一视野内完成浏览、缩放、滤波预览、重参考、坏道、坏段、事件检查。
3. 所有操作只生成记录或派生结果，不破坏原始 EEG。
4. 操作可撤销、可恢复、可保存。
5. 保存后，分析方法基于已确认的数据准备方案运行。

## 3. 范围

### 3.1 本期必须完成

- 数据准备页形成“波形 + 操作区”同屏工作台。
- 选中数据后自动加载波形，不要求用户再点击“运行质控预览”。
- 波形支持：上一段、下一段、缩放、复位、起点、窗口长度、增益、通道数量。
- 波形支持鼠标拖拽选段。
- 波形支持滚轮平移，Ctrl/Cmd + 滚轮缩放。
- 滤波预览与波形联动，显示原始/滤波预览状态。
- 重参考设置位于数据准备页，不作为分析方法。
- 坏道、坏段、标签修改在波形附近操作，并可恢复。
- 数据准备状态、选中片段、滤波/参考设置要有用户可读摘要。
- 教学模式下内置数据不可删除，并可直接完成端到端试跑。

### 3.2 本期不做

- 不做癫痫源码工作台复刻。
- 不做临床诊断、治疗建议或医疗结论。
- 不把 QC、重参考包装成分析方法。
- 不改 router / Headroom / gateway / IPC / model route。

## 4. 用户故事

### US-01 自动预览

作为科研人员，我在项目里单击 EEG 数据后，希望数据准备页自动显示波形，这样我不用理解“质控预览任务”这样的开发概念。

验收：选择数据后，`#eegCanvas` 出现非空波形；页面没有要求上传新数据。

### US-02 波形浏览

作为科研人员，我希望在数据准备页平移、缩放和调整增益，这样我可以检查不同时间段和不同通道。

验收：按钮、输入框、滚轮均能改变当前波形窗口；状态摘要显示当前窗口。

### US-03 看着波形做预处理

作为科研人员，我希望滤波、重参考、坏道、坏段操作就在波形旁边，这样每个决定都有可视依据。

验收：波形区和预处理面板在同一页面首屏内；点击快捷入口能定位到对应控件。

### US-04 可恢复

作为科研人员，我希望坏道和剔除片段都可以恢复，避免误操作。

验收：剔除片段后出现红色高亮；恢复后记录恢复动作；坏道标记和恢复都有摘要。

### US-05 教学沙盒

作为首次试用客户，我希望完成教学引导后仍留在教学模式，并用内置数据直接跑完整流程。

验收：教学模式完成引导后仍显示教学 banner；PSD 等分析任务提交使用内置项目和内置数据，不触发上传。

## 5. 信息架构

数据准备页应按以下顺序组织：

1. 当前项目和数据状态。
2. 交互式波形工作台。
3. 波形浏览工具条。
4. 波形画布、事件、状态摘要。
5. 片段、标签、坏道操作。
6. 预处理设置：滤波、重参考、参考通道、双极通道对。
7. 保存和确认数据准备。

## 6. 边界文案

- 原始 EEG 不会被直接改写。
- 滤波预览只作用于当前显示窗口。
- 重参考属于预处理，不是分析方法。
- QC 是数据准备辅助检查，不是独立分析方法。
- CSD 电流源密度计算可以作为分析/派生计算；重参考设置留在预处理。

## 7. 验收标准

### P0

- 教学数据进入数据准备页后自动显示波形。
- 波形可以拖拽选段、剔除、恢复。
- 滤波预览开关可触发预览状态变化。
- 重参考设置在数据准备页可见。
- 分析方法不再要求用户上传教学数据。

### P1

- 同屏显示波形与预处理面板。
- 操作摘要清晰说明当前窗口、选段、坏道、滤波和参考状态。
- E2E 记录截图和 JSON 证据。

### P2

- 窄屏下预处理面板下移但仍在波形后紧邻位置。
- 所有控件具备可识别选择器，便于持续测试。
