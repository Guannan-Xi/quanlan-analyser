# QLanalyser 项目清洁、波形操作与预处理同屏详细设计

日期：2026-06-26
依赖需求：`docs/product/qlanalyser_project_cleanup_waveform_preprocessing_requirements_20260626.md`

## 1. 设计目标

本设计把当前问题拆成三条产品链路：

1. 项目管理链路：用户只看到自己能继续工作的项目，内部测试资产默认不进入客户视野。
2. 波形预览链路：选择数据后自动展示可操作波形，并可放大、缩小、平移、调增益、调通道、预览滤波。
3. 预处理链路：重参考、滤波、坏道、片段和标签修改围绕波形完成，确认后形成可追溯数据准备记录。

## 2. 当前机制与根因

### 2.1 项目列表

当前 `/api/projects` 会返回本地历史验收项目。实测当前服务返回 `total=538`，其中内部/验收关键词命中约 `165`，自动 pilot 生成项目约 `319`。前端已有 `filteredWorkspaceProjects()`，但内部项目识别规则不够强，且 `pilot-user` 重复项目没有被默认隐藏。

根因：测试数据与用户项目共用同一项目列表，前端默认视图没有建立“客户视图”和“内部视图”的分层。

### 2.2 波形预览

后端 `eeg_core/preprocess/qc_preview.py` 已支持：

1. `preview.start_sec`
2. `preview.duration_sec`
3. `preview.channel_limit`
4. `filter_preview.bandpass`
5. `filter_preview.notch`
6. `waveform_preview.json`
7. `filter_preview.json`

前端 `drawEegWaveformPreview()` 当前能画波形，但窗口平移/缩放和后端参数没有完整打通；`renderEegPreviewMetadata()` 会展示任务状态和文件数量，导致用户看到内部过程而不是操作依据。

根因：后端能力存在，前端只做了初始预览，没有把控件状态变成正式 preview 请求参数。

### 2.3 预处理

确认数据准备时前端读取：

1. `#presetPrepReference`
2. `#presetPrepNotch`
3. `#presetPrepLfreq`
4. `#presetPrepHfreq`

但这些控件没有在波形旁形成明确工作台。坏道保存目前要求已确认方案，导致用户无法先看波形再标记坏道。

根因：预处理被建模成“提交前表单”，而不是“基于波形证据的编辑会话”。

## 3. 数据与状态模型

### 3.1 前端状态扩展

在 `eegState` 中增加：

```js
filteredData: null,
showFiltered: false,
filterEnabled: false,
filterLfreq: 1,
filterHfreq: 40,
filterNotchEnabled: true,
filterNotch: 50,
lastPreviewParameters: null,
```

用途：

1. `data` 保存 raw preview。
2. `filteredData` 保存后端 `filter_preview.json`。
3. `showFiltered` 控制当前画布展示 raw 还是 filtered。
4. `filterEnabled` 与低切/高切/陷波控件同步。
5. `lastPreviewParameters` 用于 E2E 和调试，证明请求参数来自当前控件。

### 3.2 预处理草稿

`prepEditState` 继续作为未保存修改记录，增加坏道操作入口：

1. `mark-bad-channel`
2. `restore-bad-channel`

这些操作只更新前端草稿和修改记录，不要求已确认方案。确认数据准备后，`confirmRealDataPreparationPlan()` 将草稿摘要写入 `qc_json` 或 `preprocessing_json`。

## 4. 项目过滤设计

新增或强化函数：

1. `isInternalOrAcceptanceProject(project)`
2. `isAutoGeneratedPilotProject(project)`
3. `projectVisibilityReason(project)`

隐藏规则：

1. 教学项目：`project.id === state.teaching.demoProjectId` 且教学模式未开启。
2. 内部项目：名称、描述、研究类型、owner 命中内部关键词。
3. 自动 pilot 项目：`owner_id === "pilot-user"` 或描述为 `QLanalyser pilot project`。
4. 归档/删除项目：普通模式隐藏。

显示规则：

1. 当前已选中项目优先保留，避免断开上下文。
2. 搜索时允许命中隐藏项目。
3. 勾选“显示内部/归档项目”时显示内部项目和归档项目。

## 5. 波形预览设计

### 5.1 请求参数

`runQcPreviewFromUi()` 改为读取当前工具栏：

```json
{
  "preview": {
    "start_sec": 0,
    "duration_sec": 10,
    "channel_limit": 8,
    "display_sfreq": 200
  },
  "filter_preview": {
    "enabled": true,
    "bandpass": {
      "enabled": true,
      "l_freq": 1,
      "h_freq": 40,
      "method": "fir"
    },
    "notch": {
      "enabled": true,
      "freqs": [50],
      "method": "fir"
    },
    "apply_to": "preview_window_only"
  },
  "boundary": "preview_only_non_diagnostic"
}
```

### 5.2 操作行为

| 控件 | 行为 | 是否请求后端 |
|---|---|---|
| 前一段 | 起点减少半个窗口，最小 0 | 是 |
| 后一段 | 起点增加半个窗口 | 是 |
| 放大 | 窗口缩短，最小 2 秒 | 是 |
| 缩小 | 窗口增加，最大 30 秒 | 是 |
| 起点输入 | 改状态；失焦或点击更新时请求 | 失焦/更新时 |
| 窗口长度 | 改状态；失焦或点击更新时请求 | 失焦/更新时 |
| 增益 | 立即重绘当前 payload | 否 |
| 显示通道数 | 立即重绘当前 payload | 否 |
| 预览滤波 | 切换 filter preview 和展示 raw/filtered | 是 |
| 复位 | 恢复默认并请求 | 是 |

### 5.3 元信息

`previewStrip` 删除为任务状态芯片，替换为 `previewStatusBar` 或简化为 4 个以内的事实项：

1. 当前窗口：`0.0-10.0 s`
2. 显示：`8 通道 / 200 Hz / uV`
3. 预览：`原始波形` 或 `滤波预览`
4. 边界：`预览不改写原始数据`

## 6. 预处理同屏设计

在 `single-file-preview-panel` 内新增 `.waveform-prep-layout`：

```text
┌──────────────────────────────────────────────┬──────────────────────────┐
│ 波形工具栏                                    │ 预处理参数               │
│ canvas                                       │ - 重参考设置             │
│ 元信息 / 事件                                │ - 低切/高切/陷波         │
│ 片段/标签/坏道快捷操作                       │ - 标记坏道/恢复          │
│ 当前修改记录                                 │ - 确认数据准备           │
└──────────────────────────────────────────────┴──────────────────────────┘
```

窄屏降级：

```text
波形工具栏
canvas
预处理参数
当前修改记录
```

底部 `preprocessing-readiness-panel` 不再放重复按钮，保留为轻量说明或删除。

## 7. CSD 与重参考边界

预处理面板：

1. 重参考设置
2. 平均参考
3. 指定通道参考
4. 双极参考

分析方法卡片：

1. CSD 电流源密度计算
2. 文案：传感器空间电流源密度/空间滤波，要求通道位置；不是源定位、诊断或脑区激活结论。

## 8. 影响范围

主要文件：

1. `frontend/index.html`
2. `frontend/app.js`
3. `frontend/styles.css`
4. `scripts/acceptance_project_cleanup_waveform_preprocessing_static.mjs`
5. `scripts/acceptance_project_cleanup_waveform_preprocessing_e2e.mjs`

后端原则：

1. 不改项目数据。
2. 不改 QC preview 核心算法，优先接现有参数。
3. 若测试发现参数未传递，再做最小后端修复。

## 9. 回滚方案

1. 项目过滤是前端显示层，可通过恢复 `filteredWorkspaceProjects()` 相关 diff 回滚。
2. 波形工具是前端控件与参数组装，可回滚到旧 `runQcPreviewFromUi()`。
3. 预处理同屏是 HTML/CSS 布局变化，不涉及数据迁移。
4. 后端历史项目不被删除，不需要数据恢复。
