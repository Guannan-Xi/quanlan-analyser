# QLanalyser 项目清洁、波形操作与预处理同屏修复需求文档

日期：2026-06-26
切片：07 full product E2E PDCA / 14_waveform_preprocessing_project_cleanup
状态：实施前需求基线

本文档是本轮修复的需求来源。实现、DeepSeek 逻辑审阅、静态检查、浏览器 E2E 和最终验收必须读取本文档；不得只依据聊天转述。

## 1. 用户反馈归纳

| 编号 | 用户观察 | 归纳问题 | 产品影响 |
|---|---|---|---|
| R1 | 项目管理中还是有很多预置项目 | 普通用户默认看到了验收、冒烟、测试、自动生成项目 | 用户无法判断哪些是自己的研究项目，产品显得像测试环境 |
| R2 | 图中没有用的可以删掉 | 波形区状态芯片混入任务号、结果文件数、内部状态 | 信息密度高但不可行动，干扰用户判断波形 |
| R3 | 波形预览要可以操作，参考最早设计，放大缩小滤波 | 前端有部分控件，但未形成完整交互闭环；滤波预览没有接到现有后端能力 | 用户看不到“调参后波形怎样变”，只能盲目确认 |
| R4 | 预处理的操作我要看着波形才能做 | 预处理按钮与波形分离，坏道/滤波/参考/确认分散在页面下方 | 科研人员需要边看信号边决定剔除、滤波和参考，否则流程不可信 |

## 2. 范围

本轮必须完成：

1. 项目管理默认只展示用户可理解、可继续工作的项目。
2. 教学模式只预置一个示例项目和一份示例数据；普通模式不显示教学项目。
3. 内部验收、冒烟、E2E、QC preview、Grouped methods、自动生成的 pilot 项目默认隐藏。
4. 项目过滤不删除后端数据，不破坏回归测试证据。
5. 波形预览具备可操作控件：前后平移、缩放时间窗、起点、增益、显示通道数、预览滤波、复位。
6. 波形预览使用后端 QC preview 已有能力：`preview.start_sec`、`preview.duration_sec`、`preview.channel_limit`、`filter_preview`。
7. 预处理控件与波形同屏显示：重参考、低切、高切、陷波、坏道标记、坏道恢复、确认数据准备。
8. 坏道剔除必须可以先做成待保存操作，并可恢复；确认数据准备后可写入审计记录。
9. CSD 电流源密度计算保留为分析方法入口；重参考、平均参考、指定通道参考、双极参考归入预处理。
10. 文案使用科研用户语言，不出现开发视角：如 task runner、preview method、beta、internal gate、fake threshold。

不在本轮做：

1. 不批量删除或归档历史测试项目。
2. 不改分析算法核心结果格式，除非 E2E 发现当前流程不可运行。
3. 不把 CSD 改造成预处理参考方案。
4. 不声称滤波已经改写原始 EEG；预览滤波只用于观察，确认方案只记录参数。
5. 不做临床、诊断、治疗或病灶定位表述。

## 3. 项目管理需求

普通模式默认列表必须满足：

1. 不显示 `proj_demo_learning` 教学项目。
2. 不显示名称、描述、研究类型或 owner 命中以下内部特征的项目：`acceptance`、`smoke`、`persistence`、`grouped methods`、`e2e`、`qc preview`、`browser qc`、`qc lab test`、`parameter exposure review`、`test`、`validator`、`dev`。
3. 不显示自动生成的 pilot 项目：`owner_id=pilot-user` 且描述为 `QLanalyser pilot project` 的重复项目。
4. 如果当前已选中项目属于内部项目，允许短暂显示当前项目，避免用户丢失上下文；但列表文案应提示它是内部/验收记录。
5. 搜索时仍可按显式输入找到项目，便于开发/验收排查。
6. “显示归档项目”应改为“显示内部/归档项目”，避免用户误解。
7. 空项目不显示虚假的阈值、准备状态或可分析结论。

验收标准：

1. `/api/projects` 返回数百个历史项目时，默认项目管理列表不出现 Acceptance、smoke、Grouped methods、QC preview、pilot-user 重复项目。
2. 开启内部/归档开关后，可以看到被隐藏项目，且开关文案明确这是内部记录。
3. 教学模式打开后只出现一个体验中心示例项目；关闭教学模式后不再出现在普通列表。

## 4. 波形预览需求

波形区必须以“科研人员正在检查当前数据”为核心，不再像任务结果摘要。

保留信息：

1. 当前数据文件名。
2. 起始时间、窗口长度、显示通道数、增益。
3. 采样率、单位、通道数。
4. 是否为预览滤波。
5. “预览不改写原始数据”的边界提示。
6. 事件/注释预览，若没有事件应清楚说明。

删除或折叠：

1. QC 任务是否生成。
2. 结果文件数量。
3. “状态：可继续准备”等空泛状态。
4. 重复的“重新加载预览”按钮，只保留波形工具栏中的主要入口。

交互要求：

1. 点击数据行后自动进入数据准备并加载波形。
2. 加载失败时显示错误原因和重试入口，不留空白画布。
3. 前/后按钮按当前窗口的一半移动预览窗口。
4. 放大/缩小按钮改变窗口长度并重新加载预览。
5. 起点和窗口长度改变后，点击“更新波形”或工具按钮应按当前参数重新请求后端。
6. 增益和显示通道数可即时重绘当前波形。
7. 预览滤波开启后，后端生成 `filter_preview.json`，前端展示滤波后波形并显示“仅预览”。
8. 复位恢复默认起点、窗口、增益、通道、滤波参数，并重新加载。

验收标准：

1. 浏览器 E2E 可点击项目数据行并看到非空 canvas。
2. 点击缩放、前后、增益、通道后，UI 状态和 canvas 像素发生合理变化。
3. 开启滤波预览并重新加载后，前端读取 `filter_preview.json`，页面显示 preview-only 边界。
4. 无数据、加载中、错误、成功状态均有可见恢复路径。

## 5. 预处理同屏需求

预处理必须从“页面下方的按钮区”改为“波形旁边/附近的工作台”。

布局要求：

1. 桌面端：波形画布占主区域，右侧或紧邻区域为预处理参数与操作。
2. 窄屏：先波形后参数，工具栏可换行，不出现横向溢出。
3. 波形元信息、预处理参数、坏道/片段/标签修改记录必须在同一视野或相邻区域内。
4. 底部重复的“数据准备检查”按钮区应删除或降级，不再与波形旁的主操作重复。

预处理参数：

1. 重参考设置：保留原始参考、平均参考、指定通道参考、双极参考。
2. 指定通道参考需要输入参考通道。
3. 双极参考需要输入通道对。
4. 低切、高切、陷波参数用于预处理方案和预览滤波。
5. 参数变更不立刻改写原始数据；确认方案后进入数据准备记录。

坏道操作：

1. 波形旁提供“标记当前通道为坏道”和“恢复最近坏道修改”。
2. 标记坏道前不要求先确认方案；先进入待保存修改记录。
3. 若已确认数据准备方案，保存/恢复按钮可写入后端 bad-channel audit。
4. 修改记录显示坏道、片段、标签三类操作及恢复数量。

验收标准：

1. 用户不向下滚动也能看到波形、滤波/参考参数、坏道标记和确认方案入口。
2. 坏道标记和恢复在确认方案前可操作并更新当前修改记录。
3. 确认方案后，坏道保存/恢复可调用后端接口。
4. 数据准备记录包含 reference、notch、l_freq、h_freq 和 preview-only 边界。

## 6. 科学与术语边界

1. “重参考设置”是预处理，不是分析方法。
2. “平均参考、指定通道参考、双极参考”都属于参考设置。
3. “CSD 电流源密度计算”可以作为分析/变换方法保留，但不能叫“参考方案”。
4. CSD 文案必须说明是传感器空间的空间滤波/电流源密度估计，不是源定位、诊断或脑区激活结论。
5. 预览滤波必须写明不改写原始数据。
