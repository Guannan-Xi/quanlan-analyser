# QLanalyser 项目工作台前端详细设计

Date: 2026-06-22
Scope: 项目管理页、项目内数据管理、项目入口后的数据准备前置页、个人中心收口、EDF 端到端测试入口。
Boundary: 不包含方法工作台、分析方法页、结果解释页、报告下载页的完整实现细则；这些能力另立设计文档。

## 1. 设计目标

本页面的目标不是“展示所有能力”，而是让用户按真实工作流完成以下任务：

1. 选择或创建项目。
2. 在当前项目内管理数据文件。
3. 进入项目后再逐步展开数据预览、预处理、标记和准备确认。
4. 通过清晰的状态反馈判断是否可以进入下一步。
5. 将账户、余额、充值、发票等非主任务内容收拢到个人中心。

前端设计必须满足三个基本原则：

- 只展示当前阶段有用的信息。
- 所有可见文案必须可被普通研究用户理解。
- 所有按钮必须对应真实动作，不允许“看得到但没用途”的伪入口。

## 2. 页面分层

### 2.1 主工作区

主工作区只保留以下四层：

1. 项目管理层
2. 当前项目数据管理层
3. 当前数据准备层
4. 当前步骤状态层

### 2.2 侧边栏

侧边栏只保留主导航和必要入口：

- 项目
- 数据文件
- 数据准备
- 分析任务
- 结果查看
- 报告下载
- 数据管理
- 费用与充值
- 发票申请
- 发票箱

其中“费用与充值 / 发票申请 / 发票箱”属于账户运营类，视觉权重要低于项目与数据主流程。

### 2.3 个人中心

个人中心固定收口以下内容：

- 当前账号信息
- 登录状态
- 退出登录
- 余额
- 充值
- 发票
- 账单
- 通知
- 安全设置
- 帮助与反馈

个人中心不得承载：

- 项目列表
- 数据列表
- 方法工作台
- 分析任务主流程
- 结果展示主区域

## 3. 默认首屏规则

### 3.1 未选中项目

页面默认只显示：

- 项目列表
- 新建项目
- 编辑项目
- 归档项目
- 删除项目
- 进入项目

不默认展示：

- 当前项目下的数据列表
- 预览区
- 预处理区
- 任务提交区
- 方法工作台
- 结果/报告详情

### 3.2 已选中项目，但未选中数据

页面显示：

- 当前项目状态
- 当前项目下的数据列表
- 上传数据
- 重命名数据
- 删除数据
- 重新上传/更新数据

不默认展示：

- 数据片段编辑面板
- 预处理确认区
- 分析任务入口
- 结果图表细节

### 3.3 已选中数据

页面才展开：

- 数据预览
- 时间窗口检查
- 通道浏览
- 坏道/坏段标记
- 注释处理
- 保存当前准备方案
- 提交到后续分析

## 4. 信息呈现规范

### 4.1 状态栏

状态栏只允许展示对用户当前决策有价值的内容，例如：

- 当前项目名称
- 当前数据文件名
- 数据是否已上传
- 数据是否已准备
- 当前是否可进入下一步
- 当前是否存在阻止项

禁止直接展示以下内容：

- 内部字段名
- 审核标签原样字符串
- 数据库状态枚举原文
- 技术备注拼接片段
- 断句怪异、对用户无帮助的内部审计短语

### 4.2 项目状态映射

项目状态不直接展示 `active / inactive / archived` 这类原始值，而应映射成业务语言：

| 内部状态 | 用户展示建议 |
| --- | --- |
| active 且已有文件 | 可进入 |
| active 但无文件 | 需上传数据 |
| archived | 已归档 |
| deleted | 已删除 |
| blocked | 暂不可用 |

### 4.3 数据状态映射

数据状态应表达任务阶段，而不是仅显示文件存在：

| 内部状态 | 用户展示建议 |
| --- | --- |
| uploaded | 已上传 |
| previewed | 已预览 |
| prepared | 已准备 |
| needs_attention | 需处理 |
| invalid | 不可用 |

### 4.4 文件备注

类似“acceptance-label · 审核备注”这种表达仅在内部审计链路中保留。
面向用户的显示应改为：

- 备注已保存
- 已更新说明
- 当前备注已应用

只有当备注会影响后续操作时，才显示简洁的自然语言说明。

## 5. 项目管理交互

### 5.1 项目列表

项目列表应支持：

- 单选当前项目
- 明显高亮当前项目
- 显示项目名、业务状态、数据概览、最后更新时间
- 进入项目详情

### 5.2 项目操作

标准操作：

- 新建项目
- 编辑项目
- 归档项目
- 删除项目
- 进入项目

操作规范：

- 编辑和归档是主动作。
- 删除必须放在二级菜单或确认层。
- 不显示无上下文意义的辅助按钮。

### 5.3 选中方式

选中项目后应满足：

- 当前选中项高亮
- 当前项目上下文立即切换
- 下方或右侧数据列表同步刷新
- 不要求用户再次“确认进入”才能看到当前项目数据

## 6. 数据管理交互

### 6.1 数据列表

数据列表只显示当前项目内的数据文件，且必须能解释每一项的业务意义：

- 文件名
- 格式
- 通道数/采样率等关键摘要
- 当前状态
- 更新时间

### 6.2 数据操作

建议保留的规范操作：

- 上传数据
- 查看
- 重命名
- 重新上传 / 更新版本
- 删除

建议避免直接暴露的用语：

- 替换文件
- 审核备注
- 内部标记字段名

如果产品内部确实需要“替换”概念，面向用户更自然的说法优先采用：

- 重新上传
- 更新文件
- 上传新版本

### 6.3 重命名规则

重命名可以是按钮，也可以是更多菜单中的动作，但必须满足：

- 位于选中数据的上下文中
- 不打扰未选中的其他数据
- 输入后立即看见明确反馈

## 7. 数据准备展开规则

### 7.1 按需展开

未选中项目时，不能默认展开数据准备。

未选中数据时，不能默认展开预览、过滤、坏道、坏段、注释、提交等动作。

### 7.2 数据准备层内容

数据准备层只负责：

- 查看当前文件
- 选择时间窗
- 浏览波形
- 标记坏道
- 标记坏段
- 查看注释
- 决定是否纳入后续分析
- 保存准备方案

### 7.3 页面边界

当前页面不得混入：

- 方法工作台的展示内容
- 复杂分析方法说明
- 结果解释长文
- 报告下载样式卡片

## 8. 视觉噪声控制

### 8.1 默认隐藏项

以下内容默认不展开：

- 无法立即操作的数据列表
- 无当前项目意义的说明
- 不相关的工作台入口
- 运营与充值信息
- 未进入上下文的后续分析模块

### 8.2 文案要求

文案必须满足：

- 术语自然
- 句子短
- 不使用内部短语拼接
- 不使用“看起来专业但不好懂”的断句

## 9. EDF 端到端测试要求

### 9.1 必测路径

必须验证以下真实路径：

1. 上传 EDF
2. 选择项目
3. 看到当前项目数据
4. 选择数据
5. 打开数据预览
6. 进行数据准备
7. 保存准备方案
8. 提交下一步分析
9. 查看结果
10. 下载报告

### 9.2 反馈要求

每一步都必须有：

- 成功反馈
- 失败反馈
- 阻止反馈

不能出现：

- 点击无反应
- 页面空转
- 状态假更新
- 只有 API 成功但 UI 无感知

## 10. 个人中心建议结构

### 10.1 必须保留

- 账号信息
- 余额
- 充值
- 发票申请
- 账单记录
- 通知
- 安全设置
- 帮助与反馈

### 10.2 可选增强

- 默认项目
- 显示偏好
- 登录设备
- 常用联系人

### 10.3 不应放入

- 项目管理
- 数据管理
- 方法工作台
- 分析主流程

## 11. 前端验收标准

- 未选项目时，页面只显示项目级信息。
- 选中项目后，才显示该项目的数据。
- 选中数据后，才显示预览与准备动作。
- 状态文案有业务意义且可理解。
- 内部审计词不裸露给用户。
- 方法工作台不在当前页首屏抢占注意力。
- 账户运营信息收口到个人中心。
- EDF 端到端测试能真实跑通。
