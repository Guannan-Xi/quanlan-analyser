## 12. Pass/Fail Rules

Full pass requires:

- all docs saved and mojibake-free
- synthetic fixture generated and manifest valid
- page/API/method inventory complete
- all required methods run or have exact blocker
- method source comparison matrix complete
- main workbench E2E completed with all direct actions
- report ZIP inventory and claim scan passed
- scientific figure audit passed
- UI screenshot/scroll/color review has no P0
- DeepSeek review completed or explicitly blocked with replacement evidence and user-visible status

Blocked if:

- any P0 remains unresolved
- a required analysis method cannot run and no safe fallback exists
- report/export claims unsafe scientific/clinical conclusions
- DeepSeek route fails when final researcher-logic review is required for acceptance
- local service cannot be reached after two bounded attempts

## 13. Execution Order

1. Preflight.

No later step may erase earlier evidence. A failure creates a repair packet or blocked receipt.
