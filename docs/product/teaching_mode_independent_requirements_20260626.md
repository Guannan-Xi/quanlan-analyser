# QLanalyser 教学模式独立产品形态需求文档

生成时间：2026-06-26T10:35:39.972886+00:00
文档状态：实施前需求冻结稿 v1
适用范围：QLanalyser Online 07 主干；02 癫痫 ML/工作台成果作为上游交接输入，不在本需求内重复实现。

## 1. 背景

用户明确提出：除“引导式教学”外，**教学模式和普通模式必须是独立的产品模式**，通过清晰按钮切换。客户看完教学模式卡片引导后，应能使用内置、不可删除的教学数据，端到端跑通每个上线分析方法。

当前 07 已有基础：

- 顶部已有“教学模式”按钮。
- 已有 `proj_demo_learning` / `eeg_demo_teaching_oddball` 教学 Oddball 数据。
- 14 号切片已验证普通模式隐藏内部/教学项目，教学模式只暴露一个教学 EEG 文件；波形预览、同页预处理、坏道保存/恢复已通过针对性 E2E。
- 02 最新进度显示癫痫 ML/工作台正在独立推进，已有预置癫痫 ML fixture、`epilepsy-workbench`、ML runner/E2E 等进展；07 侧必须先读 02 交接，再接收，不重复造第二套。

## 2. 产品目标

建立两套互不污染的用户工作空间：

1. **普通模式**：客户真实项目、真实上传数据、真实分析任务、真实报告和导出；不展示教学卡片、教学项目、内部 fixture、实验室说明。
2. **教学模式**：只面向学习和试运行；提供不可删除的教学项目与教学数据；用卡片引导、蒙版步骤提示和内置数据帮助用户端到端跑通所有上线方法。

一句话验收标准：

> 用户可通过顶部按钮在普通模式和教学模式之间切换；普通模式不出现教学项目/教学数据，教学模式内置不可删除教学数据并能端到端跑通全部上线分析方法；每个步骤有面向科研用户的引导、状态反馈、错误恢复和结果解释；所有证据通过浏览器 E2E、后端 API、视觉评审和科学绘图规范检查。

## 3. 用户角色与使用场景

| 角色 | 场景 | 主要诉求 |
|---|---|---|
| 首次试用客户 | 看完卡片引导后想快速跑通平台 | 不上传自己的数据也能完整体验 |
| 科研人员 | 想理解 PSD/TFR/PAC/癫痫筛查等方法流程 | 知道何时用、怎么看、不能过度解释什么 |
| 学生/培训用户 | 跟随教学数据学习 EEG 分析 | 有步骤提示、术语解释、错误反馈 |
| 销售/交付人员 | 演示平台能力 | 可重复、不可误删、不会污染客户真实项目 |
| 内部验收人员 | 跑全量方法回归 | 固定 fixture、固定预期输出、可比对证据 |

## 4. 模式边界

### 4.1 普通模式

普通模式必须满足：

- 只显示客户创建/上传/授权可见的项目和数据。
- 默认不显示教学项目、教学数据、验收项目、内部 fixture、归档项目。
- 不出现“教学模式下的下一步卡片”或“示例数据已准备好”等文案。
- 删除、归档、编辑操作只作用于普通项目；不能影响教学数据。
- 若用户未上传数据，不显示虚假的阈值、状态、分析结果或数据质量信息。

### 4.2 教学模式

教学模式必须满足：

- 通过顶部按钮进入，按钮文案建议：`进入教学模式` / `返回普通模式`。
- 进入后显示教学模式状态条，说明：当前使用内置教学数据，结果仅用于学习和试运行。
- 自动加载不可删除教学项目和教学数据。
- 教学项目和数据不出现在普通模式列表。
- 教学数据不可删除、不可归档、不可覆盖；删除按钮隐藏或禁用并说明原因。
- 教学模式允许用户运行分析任务、查看结果、导出教学结果包；但教学结果应带 `teaching_mode=true` metadata，不混入普通项目报告。
- 教学模式不应叫“demo 后端”“fixture”“runner”“workflow”等开发术语。

## 5. 教学数据要求

### 5.1 数据集分层

教学模式至少应支持两类教学数据：

| 数据集 | 用途 | 覆盖方法 | 状态 |
|---|---|---|---|
| Oddball 教学 EEG | 数据准备、QC、ERP、PSD、Band Power、TFR、Multitaper、PAC 的基础教学 | 大部分通用分析 | 07 已有基础 |
| 癫痫样事件教学 EEG | 癫痫 STD/ML 候选事件与复核工作台教学 | 癫痫分析 | 等 02 最终交接后接收 |

### 5.2 不可删除规则

- 教学数据记录必须有字段：`is_teaching_dataset=true`、`protected=true`、`delete_policy=not_allowed`。
- 前端不展示“删除”主按钮；如需展示禁用按钮，必须提示：`教学数据用于引导练习，不能删除。`
- 后端删除接口必须拒绝教学数据删除，返回机器可读错误：`TEACHING_DATASET_PROTECTED`。
- 管理员维护入口若未来开放，必须另走内部受保护流程，不在客户界面出现。

## 6. 引导式教学需求

教学模式不是单一欢迎页，而是任务内引导。需要两层引导：

### 6.1 卡片式学习路径

进入教学模式后显示学习路径卡片：

1. 认识数据：查看项目和教学数据。
2. 查看波形：打开波形预览，学习缩放、增益、通道数、滤波预览。
3. 数据准备：标记/恢复坏道，理解预处理方案。
4. 运行基础分析：PSD、Band Power、ERP。
5. 运行时频分析：TFR、Multitaper PSD/TFR。
6. 运行高级分析：PAC、Connectivity/CSD（若上线）。
7. 运行癫痫样事件筛查：STD/ML 入口按 02 接收状态展示。
8. 查看结果：图、表、参数、方法说明、下载。
9. 导出教学报告：导出带教学标识的结果包。

### 6.2 蒙版 + 步骤提示

必须提供可跳过、可恢复的蒙版式步骤引导：

- 引导层不遮挡当前操作目标。
- 每一步指向明确 UI 元素：项目按钮、数据卡片、波形预览、参数区、运行按钮、结果预览、下载按钮。
- 提供：上一步、下一步、跳过本次、稍后继续、退出教学。
- 键盘可操作：Esc 关闭、Tab 顺序合理、焦点不丢失。
- 不使用自动滚动造成晕动；若必须滚动，需可感知且支持 reduced motion。
- 引导进度需持久化：用户刷新后可继续或重置。

## 7. 分析方法覆盖范围

教学模式端到端至少覆盖今日上线目标：QC、PSD、Band Power、ERP、TFR、Multitaper PSD、Multitaper TFR、PAC、CSD、Connectivity、Epilepsy STD/ML。

癫痫 ML 和癫痫工作台由 02 交接后接入；教学模式不能 fork 第二套癫痫逻辑。

## 8. 文案要求

客户可见文案必须：

- 面向科研用户，不使用开发者词：`workflow`、`runner`、`fixture`、`artifact`、`backend`、`alias`。
- 不写 beta、preview、交付前复核、内部验证等上线前痕迹。
- 方法文案必须回答：这个方法看什么、适合什么问题、输入要求是什么、结果不能说明什么。
- 癫痫相关统一写：`癫痫样事件候选筛查`、`候选事件`、`科研复核`；禁止写诊断、确诊、治疗、临床决策、发作确认。

## 9. 状态与错误反馈

每个关键任务必须覆盖：空状态、加载状态、长任务状态、错误状态、成功状态、禁用状态。错误不得泄露本地绝对路径、密钥、内部账号。

## 10. 统计、科学图表与解释边界

教学模式输出图表仍需符合科学绘图规范：EEG 波形标通道/时间/单位；PSD/Band Power 标频率和频带定义；TFR 标 baseline 和色条；PAC/Connectivity 显示统计限制；癫痫只显示候选事件和复核状态。

## 11. 验收标准

### 11.1 模式隔离

- 普通模式项目列表不出现教学项目和教学数据。
- 教学模式只显示教学项目/教学数据和教学任务历史。
- 切回普通模式后，不保留教学数据选中态。
- 普通模式上传数据不进入教学模式默认数据列表。

### 11.2 教学数据保护

- UI 无法删除教学数据。
- API 删除教学数据返回 `TEACHING_DATASET_PROTECTED`。
- 教学数据刷新、重启服务后仍可用。

### 11.3 引导流程

- 首次进入教学模式显示卡片路径。
- 每张卡片可启动对应步骤。
- 蒙版引导可前进、后退、跳过、恢复。
- Esc 能关闭，焦点回到触发按钮。

### 11.4 全方法 E2E

- 每个教学方法从卡片入口启动，能选择内置教学数据，运行成功，进入结果预览，图表/表格可见，可下载。
- 失败时有可恢复错误，不出现 `[object Object]`。
- 合成虚拟数据和内置教学数据都能跑通。

### 11.5 视觉与可访问性

- 桌面、窄屏均无横向溢出。
- 蒙版不遮挡目标控件。
- 对比度、焦点环、键盘路径可用。
- 普通模式和教学模式有明确但克制的视觉区分。

## 12. 非目标

本需求不要求：重新实现癫痫 ML、创建第二套教学专用算法、普通模式展示教学项目、改 router / Headroom / IPC / gateway。
