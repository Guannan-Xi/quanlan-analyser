# QLanalyser Online 项目协作规则

## 项目名称

QLanalyser Online

## 品牌

全澜脑科学®  
QuanLan BrainScience®

## 产品定位

清晰管理 EEG 数据，规范执行分析流程，稳定交付可复核研究结果。

## 平台边界

本平台用于科研数据管理与分析辅助，结果不作为临床诊断依据。

## 当前目标

QLanalyser Online v0.1 Pilot：稳定 MVP，用于客户免费试用，不是完整商业化平台。

## 核心工作流

1. 登录
2. 创建项目
3. 上传 EEG 文件
4. 创建分析任务
5. 后台运行分析
6. 查看任务状态
7. 查看结果
8. 下载报告包

## 优先功能

- 登录页
- 工作台首页
- 项目管理
- EEG 文件上传
- 文件元数据查看
- Metadata/QC 分析
- Resting PSD 分析
- 分析任务状态
- 任务失败原因展示
- HTML 报告
- ZIP 报告包下载
- 管理员入口
- 基础日志
- 基础部署和备份说明

## 开发规则

- 不要重写整个项目。
- 不要一次性大规模重构。
- 每次只完成一个明确小任务。
- 不要引入不必要的新依赖。
- 不要删除现有可用功能，除非明确说明原因并等待确认。
- 产品名统一为 QLanalyser Online。
- 不要残留 QLanalyser EEG。
- 原始 EEG 文件不要存入数据库，数据库只保存文件元数据和路径。
- 长时间 EEG 分析任务不应阻塞 HTTP 请求。
- 分析结果应保存输入文件信息、分析参数、运行日志、软件版本、输出文件路径。
- 每次任务完成后必须更新 docs/PROJECT_STATUS.md 和 docs/TASK_LOG.md。
- 如有产品或架构决策变化，必须更新 docs/DECISIONS.md。

## AI 协作分工

ChatGPT 网页版负责产品方向、调研、学习、任务拆解和评审。  
Codex 负责代码阅读、实现、测试和交接。

Codex 不应假设自己掌握全部产品背景。  
如果遇到产品定位、技术架构、数据库结构、EEG 分析方法、安全权限、部署策略等重大问题，应提醒我先回到 ChatGPT 网页版讨论确认。

每次任务完成后，Codex 必须提醒我将以下内容同步给 ChatGPT 网页版：

- docs/PROJECT_STATUS.md 最新内容
- docs/TASK_LOG.md 最近一次记录
- 本次修改文件列表
- 本次测试结果
- 当前报错或风险点
- Codex 建议的下一步任务
